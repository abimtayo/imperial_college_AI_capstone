Motivation 
---------------------------------------------------------------------
The data set was created to support a black box Bayesian optimisation challenge. The data captures both inputs to- and outputs from the black box optimiser. Data is available for 8 surrogate functions and while the inputs to the functions vary, the output is consistently a single value.
Composition: what does it contain? What is the size and format and are there any gaps?

The data consists of two sets: inputs and outputs. Both sets are used for optimising surrogates for individual functions. The inputs vary as follows:
	- Functions 1 and 2: Two input points
	- Function 3: Three input points
	- Functions 4 and 5: Four input points
	- Function 6: Five input points
	- Function 7: Six input points
	- Function 8: Eight input points

Composition
---------------------------------------------------------------------
The data is stored in Comma Separated Variable format (csv). For the inputs the columns named and stored in function and point order. For example, F1X1 represents Function 1, point 1. The output data set contains a single column for each function per week, giving 8 columns in total.

The data is stored in time order, with each row representing the data points for a particular week.

Collection Process
----------------------------------------------------------------------

The data is generated by use of function surrogate models and acquisition functions. Point in time data sets are ingested from the CSV files and Bayesian otpimisation processes for each surrogate attempt to fit a model to the data. 

Acquisition functions are used to  provide suggested points for the current round of the challenge. These points are then uploaded to the black box optimisation portal.
After a few days, the portal returns the actual output values generated. The submit inputs and generated outputs are then added to the relevant data set as the points for the given week. 

So far data has been generated over 10 weeks in addition to initial data points that were provided as part of the challenge.
A range of strategies are used to generate the point suggestions, including:
  - Surrogate model selection and utilisation, i.e. gaussain processes
  - Hyperparameter tuning
  - Automatic Relevance Detection (ARD) 

Preprocessing and Uses
----------------------------------------------------------------------
Data inputs are transformed from individual values into a single data set that grows as new data arrives. This simplifies processing of the data. The suggested points are constrained to 6 decimal places and are separated by a “-“.

The data is intended for function simulation based on the available data at a point in time. IT is not intended to be ab exhaustive or representative set of inputs and outputa for each function. 

The ouputs are actual outputs for the given points but the points are patchy and only represent a portion of the input space.
The data is also not suitable for extrapolating unseen values and not warranty is made as to its degree of completeness.

Distribution and maintenance
----------------------------------------------------------------------
The data is available in the current GitHub repository. The original format of the input data is retained in separate per-function folders and weekly iterations of all data is contained in separate files. The full data sets will be updated before the end of the competition. 

There are no terms of use beyond the guidance provided in the previous section.
